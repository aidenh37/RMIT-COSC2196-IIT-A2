Tools

2 main tools have been utilised in our group work on this report.
GitHub and MS (Microsoft) Teams.

Initially use of these tools was slow, as team members learned the new tools, understanding their processes, and how to utilise them, and continued work on their individual assigned tasks.

GitHub has been used exclusively for the webpage component of the report, while MS teams has been exclusively used for the development of the Pdf report format. Using the 2 different tools to complete separate components of the overall assessment, has helped reduce confusion, and work duplication. Another benefit has been to produce 2 distinct audit trails. Having 2 distinct records of participation, creates a simplified data trail, and reduced duplication of audit data.

Aiden Howlett, setup the initial GitHub repository, and Robert Longmore was the significant contributor to the groups GitHub website, as this was his chosen body of work to contribute to the group assessment. As a result, the GitHub commit data is significantly influenced by this one contributor.

As the PDF document component of the assessment was created and collaboratively worked on via MS teams, and each team member contributing equally to this document. The contributor data on the use of MS Teams varies significantly, to reflect the contributions of each member as they completed their components of the assessment. Robert Longmore had the smallest contribution here, mainly small edits and updates on the progress of the website development to ensure it reflects the Pdf document accurately, and to solicit team feedback on his work.

MS Teams document provided not only a record of contributors to the document, but also provided an avenue for continued feedback and discussion between contributors as the work progressed. This might have some influence on the meeting agendas, as a lot of hurdles and discussion was via the MS Team workspace, resulting in solutions and agreed changes. Requiring less time to be taken via the scheduled meetings.
